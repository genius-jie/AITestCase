# Prompt设计指南

## 1. Prompt设计原则

### 1.1 清晰明确
- 目标明确，避免模糊
- 指令清晰，易于理解
- 范围明确，避免歧义

### 1.2 结构合理
- 层次分明，逻辑清晰
- 重点突出，主次分明
- 格式规范，易于阅读

### 1.3 具体详细
- 提供足够的上下文
- 明确输出格式要求
- 包含必要的示例

### 1.4 可执行性强
- 指令可操作，可验证
- 避免抽象概念
- 提供明确的步骤

## 2. Prompt核心结构

### 2.1 角色定义
- 明确Agent的身份
- 定义Agent的专业领域
- 描述Agent的核心能力

### 2.2 任务描述
- 明确任务目标
- 定义任务范围
- 说明任务要求

### 2.3 输出要求
- 输出格式规范
- 输出内容要求
- 输出质量标准

### 2.4 约束条件
- 明确限制条件
- 定义边界范围
- 说明禁止行为

### 2.5 示例（可选）
- 提供正面示例
- 说明错误示例
- 展示期望输出

## 3. Prompt设计流程

### 3.1 需求分析
- 明确任务目标
- 分析用户需求
- 确定输出要求

### 3.2 结构设计
- 设计Prompt的整体结构
- 确定各部分的内容
- 规划各部分的顺序

### 3.3 内容编写
- 编写角色定义
- 描述任务要求
- 明确输出格式
- 定义约束条件
- 添加示例（可选）

### 3.4 测试优化
- 测试Prompt的效果
- 分析输出结果
- 优化Prompt内容
- 调整结构和表述

## 4. Prompt类型

### 4.1 指令型Prompt
- 直接给出明确的指令
- 适用于简单任务
- 输出格式固定

### 4.2 示例型Prompt
- 提供示例引导输出
- 适用于复杂任务
- 输出质量较高

### 4.3 迭代型Prompt
- 支持多轮交互
- 适用于复杂决策任务
- 可以逐步优化输出

## 5. Prompt最佳实践

### 5.1 使用清晰的语言
- 避免使用模糊词汇
- 避免使用复杂句式
- 使用专业术语

### 5.2 提供足够的上下文
- 包含必要的背景信息
- 说明任务的上下文
- 提供相关的参考资料

### 5.3 明确输出格式
- 指定输出的格式类型
- 提供输出的结构示例
- 说明输出的详细要求

### 5.4 定义清晰的约束
- 明确限制条件
- 说明禁止行为
- 定义边界范围

### 5.5 测试和优化
- 多次测试Prompt效果
- 收集反馈并优化
- 持续改进Prompt质量

## 6. Prompt设计示例

### 6.1 简单指令型Prompt
```
你是一位专业的测试用例生成专家，请根据以下需求生成测试用例：

需求：用户登录功能，支持用户名和密码登录，支持忘记密码功能。

输出格式：
| 测试用例ID | 测试场景 | 输入 | 预期输出 |
|------------|----------|------|----------|
| TC001      |          |      |          |
```

### 6.2 示例型Prompt
```
你是一位专业的文案写作专家，请根据以下示例生成产品介绍文案：

产品：智能手表

示例：
智能手表，你的贴身健康管家。24小时监测心率、睡眠质量，支持多种运动模式，让你随时掌握健康状况。

输出要求：
- 突出产品的核心功能
- 语言简洁流畅
- 富有吸引力
```
