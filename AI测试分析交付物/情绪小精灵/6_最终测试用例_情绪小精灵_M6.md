# 情绪小精灵 - M6灯光交互系统测试用例

## 模块6：灯光交互系统

### 功能点：6.1.1 情绪灯光

- [正例]快乐情绪灯光显示_P0  
    - 操作步骤：1. 唤醒小精灵 2. 用户表达快乐情绪（如："今天真开心！"） 3. 观察小精灵灯光变化  
        - 预期结果：小精灵显示快乐情绪对应的灯光（如明亮的黄色或橙色），灯光平滑过渡，与情绪匹配度高

- [正例]情绪灯光与语音同步_P0  
    - 操作步骤：1. 唤醒小精灵 2. 用户表达不同快乐情绪（如："哈哈，太好笑了！"） 3. 观察灯光与语音回应的同步性  
        - 预期结果：灯光变化与语音回应同时开始，同时结束，节奏协调一致

- [反例]无效情绪输入灯光响应_P0  
    - 操作步骤：1. 唤醒小精灵 2. 用户输入无意义字符串（如："asdfghjkl"） 3. 观察灯光变化  
        - 预期结果：小精灵灯光保持默认状态，不显示情绪灯光效果

- [边界]情绪表达边界情况灯光响应_P0  
    - 操作步骤：1. 唤醒小精灵 2. 用户以极低音量表达快乐情绪 3. 观察灯光变化  
        - 预期结果：小精灵仍能正确识别情绪并显示对应的快乐情绪灯光

- [正例]多种快乐情绪场景灯光变化_P1  
    - 操作步骤：1. 唤醒小精灵 2. 用户分别表达不同场景的快乐（如："考试通过了！"、"收到礼物了！"） 3. 观察灯光差异  
        - 预期结果：不同快乐场景下灯光有细微差异，如亮度或闪烁频率不同，但均保持快乐情绪基调

- [正例]情绪灯光持续时间_P1  
    - 操作步骤：1. 唤醒小精灵 2. 用户表达快乐情绪 3. 计时灯光持续时间  
        - 预期结果：灯光效果持续时间与语音回应长度匹配，不超过10秒

- [反例]负面情绪输入快乐灯光_P1  
    - 操作步骤：1. 唤醒小精灵 2. 用户表达负面情绪（如："今天真难过"） 3. 观察灯光变化  
        - 预期结果：小精灵不显示快乐情绪灯光，可能显示其他情绪灯光或保持默认状态

- [边界]快速连续情绪输入灯光响应_P1  
    - 操作步骤：1. 唤醒小精灵 2. 用户快速连续表达多次快乐情绪 3. 观察灯光变化  
        - 预期结果：小精灵能正确切换灯光效果，无卡顿或延迟，每次情绪输入都有对应的灯光响应

- [正例]情绪灯光颜色准确性_P2  
    - 操作步骤：1. 唤醒小精灵 2. 用户表达快乐情绪 3. 检查灯光颜色是否符合快乐情绪定义  
        - 预期结果：灯光颜色为明亮的黄色或橙色，无明显偏差

- [反例]非情绪类指令灯光响应_P2  
    - 操作步骤：1. 唤醒小精灵 2. 用户发出非情绪类指令（如："现在几点？"） 3. 观察灯光变化  
        - 预期结果：小精灵不显示情绪灯光，可能显示状态灯光或保持默认状态

- [操作体验性]情绪灯光视觉舒适度_P2  
    - 操作步骤：1. 唤醒小精灵 2. 用户表达快乐情绪 3. 观察灯光是否刺眼或闪烁过快  
        - 预期结果：灯光亮度适中，闪烁频率舒适，无视觉疲劳感

### 功能点：6.1.2 状态灯光

- [正例]待机状态灯光显示_P0  
    - 操作步骤：1. 小精灵处于待机状态 2. 观察灯光状态  
        - 预期结果：小精灵显示待机状态对应的灯光（如微弱的蓝色呼吸灯），节奏缓慢，功耗低

- [正例]唤醒状态灯光变化_P0  
    - 操作步骤：1. 小精灵处于待机状态 2. 唤醒小精灵（如："小精灵"） 3. 观察灯光变化  
        - 预期结果：小精灵从待机灯光平滑过渡到唤醒状态灯光（如明亮的白色），响应及时

- [反例]异常状态灯光错误显示_P0  
    - 操作步骤：1. 小精灵处于正常状态 2. 人为制造异常（如断开网络） 3. 观察灯光变化  
        - 预期结果：小精灵显示异常状态对应的灯光（如红色闪烁），准确反映当前状态

- [边界]低电量状态灯光提示_P0  
    - 操作步骤：1. 小精灵电量低于10% 2. 观察灯光状态  
        - 预期结果：小精灵显示低电量状态灯光（如红色呼吸灯），清晰提示用户充电

- [正例]充电状态灯光变化_P1  
    - 操作步骤：1. 小精灵电量较低 2. 连接充电器 3. 观察灯光变化  
        - 预期结果：小精灵显示充电状态灯光（如绿色渐变），充电完成后灯光变为满电状态

- [正例]工作状态灯光指示_P1  
    - 操作步骤：1. 唤醒小精灵 2. 发出指令（如："播放音乐"） 3. 观察工作过程中灯光变化  
        - 预期结果：小精灵显示工作状态灯光（如白色闪烁），清晰指示正在处理指令

- [反例]状态灯光与实际状态不符_P1  
    - 操作步骤：1. 小精灵处于正常工作状态 2. 观察灯光状态  
        - 预期结果：灯光状态与实际工作状态一致，无错误指示

- [边界]网络状态切换灯光响应_P1  
    - 操作步骤：1. 小精灵连接网络 2. 断开网络 3. 重新连接网络 4. 观察灯光变化  
        - 预期结果：网络状态变化时，灯光能准确反映，切换过程平滑无卡顿

- [正例]状态灯光颜色区分度_P2  
    - 操作步骤：1. 让小精灵依次进入不同状态（待机、唤醒、工作、异常） 2. 观察灯光颜色差异  
        - 预期结果：不同状态的灯光颜色有明显区分，用户能轻松识别当前状态

- [反例]无状态变化时灯光异常_P2  
    - 操作步骤：1. 小精灵处于稳定状态 2. 观察5分钟内灯光变化  
        - 预期结果：灯光保持稳定，无异常闪烁或颜色变化

- [操作体验性]状态灯光直观易懂_P2  
    - 操作步骤：1. 让不同用户观察小精灵在不同状态下的灯光 2. 询问用户对灯光含义的理解  
        - 预期结果：80%以上用户能正确理解各状态灯光的含义，无需额外说明
